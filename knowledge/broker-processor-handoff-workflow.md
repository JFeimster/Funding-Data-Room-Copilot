# Broker and Processor Handoff Workflow

## Workflow Objective

Create a predictable path from initial intake to a human-reviewed handoff. The workflow should reduce repeated requests, unclear ownership, missing context, and untracked client follow-up.

The workflow must not automate final submission without a human checkpoint.

## User Roles

### Business Owner

Provides documents, confirms facts, and approves explanations.

### Broker or Deal Owner

Manages the relationship, clarifies the request, and owns the next commercial step.

### Processor

Tracks documents, identifies gaps, prepares the package, and maintains status.

### Referral Partner

Provides initial context and hands the relationship to the authorized funding team.

### Bookkeeper or CPA

Supports financial records and factual accounting explanations.

### Human Reviewer

Determines whether the package is ready for the intended next step.

## Recommended Stages

1. New Lead
2. Intake Started
3. Documents Requested
4. Documents Partially Received
5. Missing Documents Follow-Up
6. Context Notes Needed
7. Ready for Internal Review
8. Hold — Blocker Present
9. Partner Handoff Prepared
10. Submitted by Authorized Human
11. Questions Pending
12. Offer Review Support
13. Closed / Funded
14. Declined / Nurture
15. Renewal Watch

## Intake

Capture:

- client and business name
- user role
- requested amount
- minimum useful amount
- use of funds
- timing
- business type
- time in business
- available documents
- consent and privacy notes
- referral source

If requested amount, legal entity, or ownership conflicts, ask one focused question before creating the final snapshot.

## Documents Requested

Send a precise list rather than “send your financials.”

Include:

- document
- date range
- file format
- secure delivery method
- due date
- contact for questions

Do not imply that every provider requires the same list.

## Documents Partially Received

Processor actions:

- preserve original files
- place files in the correct folder
- update tracker status
- identify missing pages or months
- standardize working-copy names
- flag privacy issues
- avoid interpreting financial performance

## Missing Documents Follow-Up

Use:

```text
Hi [Name] — we received [items]. The file still needs [specific items] before it can move to human review. Please send [exact requirement] by [date] using [approved method]. Let us know if the document is unavailable so we can update the tracker rather than guessing.
```

Follow-up should be firm, specific, and respectful.

## Context Notes Needed

Create context requests for:

- revenue changes
- large deposits
- large withdrawals
- seasonality
- equipment purchases
- delayed receivables
- merchant reserves
- platform payout timing
- existing debt

The owner must confirm the final factual note.

## Ready for Internal Review

Minimum conditions:

- no unresolved privacy Blocker
- identity and ownership are consistent
- requested amount and use of funds are clear
- tracked documents are received or explicitly marked not applicable by a human
- context notes are reviewed
- debt schedule is current enough for the workflow
- deal snapshot is updated
- all Blockers are resolved

This stage means ready for a person to review—not approved or ready for automatic submission.

## Submission Handoff

The handoff package should include:

- deal snapshot
- document index
- missing-document tracker
- context notes
- current debt schedule
- use-of-funds summary
- questions still open
- privacy status
- human reviewer and date

Only an authorized human should make the final submission decision.

## Partner Questions

Track:

| Question | Assigned To | Due Date | Source Document | Response Status | Human Review |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

Do not manufacture an answer to avoid a delay.

## Offer Review Support

The Copilot may organize:

- offer terms supplied by the user
- questions to ask
- payment frequency
- total repayment information when clearly provided
- document comparison fields
- follow-up tasks

It must not declare the “best” option or replace qualified financial or legal review.

## Closed / Funded

Operational actions:

- record final status reported by the authorized human
- archive working versions
- preserve consent and correspondence according to policy
- schedule renewal or follow-up only when appropriate
- remove unnecessary shared access

Do not infer that a file was funded.

## Declined / Nurture

Record only the status and reason supplied by an authorized source.

Possible next actions:

- correct missing documents
- improve financial reporting
- resolve business identity issues
- review debt obligations
- update the package later
- refer to a qualified advisor

Do not promise that changes will produce approval.

## Renewal Watch

Track:

- reported funding date
- renewal or review date
- document refresh date
- updated statement period
- revenue and debt changes
- consent for follow-up

## Blocker Escalation

When a Blocker is identified:

1. Update the tracker.
2. Notify the deal owner.
3. Assign one corrective action.
4. Move the stage to `Hold — Blocker Present` when appropriate.
5. Require human confirmation before clearing it.
6. Record who cleared it and when.

## Follow-Up Ownership

Every task needs one owner. Avoid “team” as the owner unless a specific person is accountable.

Use:

- owner
- broker
- processor
- bookkeeper
- CPA
- attorney
- internal reviewer

## Human Review Gate

Before submission, confirm:

- [ ] identity and ownership
- [ ] requested amount
- [ ] use of funds
- [ ] document period
- [ ] missing-document status
- [ ] debt schedule
- [ ] context notes
- [ ] privacy review
- [ ] deal snapshot
- [ ] authorized submission decision

## Workflow QA Checklist

- [ ] Stage matches actual status
- [ ] Every open item has an owner
- [ ] No Blocker is hidden in notes
- [ ] Due dates are current
- [ ] Original files are preserved
- [ ] Provisional assumptions are labeled
- [ ] Client folders are separated
- [ ] Handoff notes are factual
- [ ] Submission was not automated without review
- [ ] Final status came from an authorized human
